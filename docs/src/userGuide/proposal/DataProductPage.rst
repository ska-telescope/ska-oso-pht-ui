Observatory Data Product Page
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Data Products are associated to an Observation which have a valid association to a target. Here, you can request a single or multiple observatory data products for an observation set added to your submission. The |icosdp| button is not active until an observation exists. 
:numref:`Figure %s <figure50>` shows the layout of the observatory data product page when no data product has been added to Observation(s).

Key Information and Actions
===========================

- View data products added to an observation set.
- Add new data products.
- Delete data products.
- If no observation is added to your submission yet, the |icosdp| button will be deactivated.


Layout and Navigation
=====================

The are two parts to the observatory data product page:

1. **Landing page**


   To add observatory data product to an observation, click the  |icosdp| button. Once a data product 
   has been associated to an Observation, the observatory data product landing page will 
   display a table as shown in :numref:`Figure %s <datapage>`. This table displays the following:

      - *"Actions"* : 
         - *Expand* : Expand to view details of the data product added to the observation set
         - *Edit" : Edit the data product parameters. This takes you to the data product management page (see next section).
         - *Delete* : Delete the data product association to an observation set.
      -"SDP Id": Unique identifier for the data product association to an observation set.
      - *Observation Type*: Type of observation associated to the Data PProduct.
      - *Observatory ID*:  Unique identifier for the Observation set.
      - *Subarray Configuration*: Subarray configuration of the observation set.
      - *Observing Band*: Observing band of the selected observation set.
      - *Frequency Range*: Frequency range of the selected observation set.

2. **Data Product Management**

- Clicking on the |icosdp| button takes the user to a page where parameters of the 
  different data product(s) can be specified as shown below:
